# Style Learning Plugin 集成说明

## 概述

设计了一个完整的"风格学习"插件，这个插件可以让AI分析整本小说并学习作者的写作风格，然后用这种风格进行创作。

## 插件功能特色

### 🎯 核心功能
- **深度风格分析**: 从词汇、句法、叙述、描写、节奏等5个维度全面分析
- **智能风格学习**: 提取并建模作者的写作特征，生成可复用的风格配置
- **风格化创作**: 使用学习的风格进行章节创作，高度还原目标风格
- **风格验证**: 自动检查创作内容的风格一致性
- **风格融合**: 支持多种风格的融合创新

### 🛠️ 技术特点
- **多维度分析**: 词汇、句式、叙述、描写、节奏全覆盖
- **量化建模**: 将定性风格特征转化为可操作的参数
- **实时反馈**: 创作过程中提供风格匹配度评估
- **专家指导**: 内置风格学习专家系统

## 主要命令

### 1. 风格分析命令
```bash
/style-analyze [文件路径或目录]    # 分析写作风格
/author-analyze [作者目录]        # 分析作者整体风格
/style-compare [样本1] [样本2]    # 对比分析风格差异
```

### 2. 风格学习命令
```bash
/style-learn [样本路径] --name=[风格名称]  # 学习目标风格
/style-list                              # 查看已学习的风格
/style-info [风格名称]                   # 查看风格详情
```

### 3. 风格化创作命令
```bash
/write-styled [章节名] --style=[风格名]     # 风格化创作
/style-convert [文件] --target=[风格名]    # 转换现有文本风格
/rewrite-styled [文本] --style=[风格名]    # 改写特定段落
```

### 4. 高级功能命令
```bash
/style-blend [风格1] [风格2] --ratio=[比例] --name=[新风格名]  # 风格融合
/style-validate [文件] --target=[风格名]                    # 验证风格一致性
/style-similarity [文本1] [文本2]                          # 计算风格相似度
```

## 集成到现有项目

### 方法1: 手动集成

1. **复制插件文件**
   ```bash
   cp -r plugins/style-learning /path/to/your/novel-writer/plugins/
   ```

2. **创建必要目录**
   ```bash
   mkdir -p samples analysis-reports memory/styles styled-output
   ```

3. **更新项目配置**
   在 `package.json` 中添加插件引用：
   ```json
   {
     "novelWriter": {
       "plugins": ["style-learning"]
     }
   }
   ```

4. **生成平台命令**
   根据你使用的AI平台（Claude、Cursor、Gemini等），将命令文件复制到对应目录。

### 方法2: 自动安装（推荐）

运行安装脚本：
```bash
node plugins/style-learning/install.js
```

安装脚本会自动：
- 检查项目环境
- 创建必要目录
- 复制插件文件
- 更新项目配置
- 生成各平台命令文件
- 创建示例文件

```bash
# 分析金庸作品风格

/style-learn samples/jinyong/ --name="金庸风格"
```

### 3. 风格化创作
```bash
# 使用金庸风格创作新章节
/write-styled 第1章 初入江湖 --style="金庸风格"
```

```bash
# 验证风格一致性
/style-validate styled-output/第1章-初入江湖-金庸风格.md --target="金庸风格"
```

## 技术实现要点

### 1. 风格特征提取
- **词汇层面**: 高频词汇、专业术语、情感词汇、修饰词使用
- **句法层面**: 句长分布、复杂度、标点使用、句式变化
- **叙述层面**: 视角选择、时态运用、叙述距离、对话比例
- **描写层面**: 感官描写、修辞手法、意象象征、细节程度
- **节奏层面**: 段落长度、信息密度、张弛节奏、过渡技巧

### 2. 风格建模
将定性特征转化为量化参数：
```json
{
  "formalityLevel": 0.8,      // 语言正式度
  "complexityScore": 0.6,     // 句式复杂度
  "detailLevel": 0.7,         // 描写详细度
  "emotionalIntensity": 0.5,  // 情感强度
  "rhythmVariation": 0.6      // 节奏变化度
}
```

### 3. 风格应用
在创作时根据风格参数调整：
- 词汇选择策略
- 句式生成规则
- 叙述技巧应用
- 描写手法使用
- 节奏控制方法
